## üöÄ Production Deployment Infrastructure

### Remote Deployment System

Deploy to production servers with automated SSH deployment:

```bash
# Configure your server
cp deploy-config/production.conf.example deploy-config/production.conf
nano deploy-config/production.conf

# Test deployment
./scripts/deploy-remote.sh production --dry-run

# Deploy to production
./scripts/deploy-remote.sh production --restart
```

**Features:**
- ‚úÖ Multi-environment support (production/staging/demo)
- ‚úÖ SSH connectivity testing
- ‚úÖ Automated pre-deployment backups
- ‚úÖ rsync-based file transfer

## üéØ Getting Started

### Quick Start (5 minutes)

```bash
# 1. Build locally
./scripts/deploy-dashboard.sh

# 2. Start server
cd server && npm install && node server.js

# 3. Open browser
# http://localhost:3001

# 4. Setup admin account
# Complete wizard, set API key
```

### Production Deployment (30 minutes)

```bash
# 1. Setup server (Ubuntu/Debian)
# Follow: PRODUCTION_DEPLOYMENT.md

# 2. Configure deployment
cp deploy-config/production.conf.example deploy-config/production.conf
# Edit with your server details

# 3. Deploy
./scripts/deploy-remote.sh production --dry-run
./scripts/deploy-remote.sh production --restart

# 4. Verify
# Check application is running
# View logs: pm2 logs systracker
```

### CI/CD Automation

```bash
# 1. Add GitHub secrets
# SSH_PRIVATE_KEY, REMOTE_HOST, REMOTE_USER, REMOTE_PATH

# 2. Push to main
git push origin main

# 3. GitHub Actions deploys automatically
# View status in Actions tab
```
