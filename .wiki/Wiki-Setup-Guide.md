## Wiki Maintenance

### Adding New Pages

1. Create `.md` file in `.wiki/` folder
2. Follow naming: `Page-Name.md` (with hyphens)
3. Add to `_Sidebar.md` for navigation
4. Push to GitHub

### Updating Existing Pages

1. Edit `.md` file
2. Commit changes
3. Push to GitHub
4. Wiki updates automatically

### Naming Conventions

- Use descriptive names: `Agent-Deployment-Testing.md`
- Use hyphens for spaces: `Quick-Start.md`
- Keep names concise but clear
- Avoid special characters

GitHub Wiki auto-generates TOC from headings:

## Best Practices

1. **Keep it Simple** - Use clear, concise language
2. **Add Examples** - Include code examples and screenshots
3. **Make it Scannable** - Use headings and lists
4. **Link Between Pages** - Help readers navigate
5. **Update Regularly** - Keep docs current
6. **Use Checklists** - For procedures and validation
7. **Organize by Task** - Not by technical detail

## Creating Wiki Locally

### Setup

```bash
# Clone repository
git clone https://github.com/Redwan002117/SysTracker.git
cd SysTracker

# Create .wiki directory
mkdir .wiki
cd .wiki

# Initialize git
git init

# Add remote (if not already there)
git remote add origin https://github.com/Redwan002117/SysTracker.wiki.git
```

### Workflow

```bash
# Create new page
echo "# New Page" > New-Page.md

# Edit file
nano New-Page.md

# Add file
git add New-Page.md

# Commit
git commit -m "Add new page: New Page"

# Push to wiki
git push origin master
```
